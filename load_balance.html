High Throughput Computing

  • Traditional HPC jobs use a lot of nodes (wide)
  • There are also workflows that consist of large number of small jobs
  • Small may mean short in duration or else involving a small number of processors
  • Workflows with many small jobs are often referred to as High Throughput Computing

System Challenges

  • Job scheduler needs to keep track of all queued jobs and run them efficiently
  • When one user submits 50,000 jobs at once, the scheduler is taxed
  • Some HPC sites allow only a fixed (small) number of jobs per user
  • There are startup and wind-down tasks for each job
  • If a job only lasts a minute, the overhead from these tasks is substantial

Possible Solutions

  • Moab, SLURM, or Condor
  • Open Science Grid
  • Bash, pbsdsh, or Falkon
  • MPI (mpi4py): not always what we want
  • Python
    • SAGA BigJob
    • multiprocessing: simple and included
    • Message broker: IPython parallel, Celery
  • And many, many more...Parallel Processing Python Wiki

The map abstraction

Python acquired lambda, reduce, filter and map, courtesy of a Lisp hacker who missed them and submitted working patches. -Guido van Rossum 
def square(x):
    return x*x

numbers = [1,2,3]

def map_squares(nums):
    res = []
    for x in nums:
        res.append( square(x) )
    return res

or... 

results = map(square, numbers)

map is the key to parallelism in python
